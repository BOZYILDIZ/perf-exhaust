## ✨ Présentation

**PERF'EXHAUST** est un atelier artisanal spécialisé dans la **fabrication d'échappements sur mesure**, basé à **Rountzenheim-Auenheim**, dans le Bas-Rhin (67), en Alsace.

Ce dépôt contient le site vitrine de l'atelier : une vitrine numérique pensée pour transmettre, dès les premières secondes, l'exigence artisanale et la précision technique qui caractérisent chaque ligne d'échappement fabriquée en atelier — soudure TIG sur inox 304L/316L, conception acoustique sur mesure, finitions premium.

> [!NOTE]
> Le site n'est pas un simple catalogue : il agit comme un **configurateur d'intention**. Chaque visiteur y trouve une réponse directe à sa question — sonorité recherchée, matériau, délai, zone géographique — que ce soit via une lecture humaine classique ou via un moteur génératif (ChatGPT, Gemini, Perplexity...).

**Pourquoi ce projet existe :**

| Contexte | Réponse apportée |
|---|---|
| 🎯 Marché de niche, très technique | Un discours de contenu précis (matériaux, procédés, sonorités) plutôt qu'un site vitrine générique |
| 📍 Clientèle locale + régionale (Grand Est) | Un maillage GEO/SEO ciblé sur les villes réellement desservies |
| 🤝 Confiance & artisanat | Une esthétique premium noire/bleue, cinématique, loin des codes "garage" traditionnels |
| 🤖 Recherche assistée par IA en plein essor | Un contenu structuré et des données `schema.org` pensés pour être cités par les moteurs génératifs |

## ⚡ Installation

> [!TIP]
> Node.js **20+** est recommandé pour une compatibilité optimale avec Next.js 16 et Turbopack.

```bash
# 1. Cloner le dépôt
git clone https://github.com/BOZYILDIZ/perf-exhaust.git
cd perf-exhaust

# 2. Installer les dépendances
npm install

# 3. Configurer les variables d'environnement
cp .env.example .env.local

# 4. Lancer le serveur de développement
npm run dev
```

Le site est accessible sur **[http://localhost:3000](http://localhost:3000)**.

## 🔐 Variables d'environnement

## 🔎 SEO

Le site applique une stratégie SEO technique complète, entièrement gérée côté code :

- **Metadata dynamique** — `title` templatisé, `description`, `keywords`, `canonical` sur chaque route via l'API `Metadata` de Next.js
- **`sitemap.xml`** — généré dynamiquement (`src/app/sitemap.ts`), couvrant les 6 pages statiques indexables + les 15 pages de réalisations
- **`robots.txt`** — autorise l'indexation complète du site, exclut `/api/` et `/_next/`
- **Open Graph & Twitter Cards** — image sociale dédiée, titres et descriptions optimisés au partage
- **Données structurées `schema.org`** (JSON-LD), injectées dans `<head>` ou en ligne selon le contexte :

  | Schéma | Usage |
  |---|---|
  | `Organization` / `LocalBusiness` | Identité de l'atelier, zone de service, catalogue d'offres |
  | `WebSite` (+ `SearchAction`) | Déclaration du site pour les moteurs de recherche |
  | `FAQPage` | Questions fréquentes structurées |
  | `Service` | Chaque prestation individuelle |
  | `BreadcrumbList` | Fil d'Ariane sur les pages profondes |
  | `CreativeWork` (projet) | Chaque réalisation de la galerie |

## 🌍 GEO — Référencement génératif

> [!NOTE]
> **GEO** (*Generative Engine Optimization*) désigne l'optimisation d'un contenu pour qu'il soit **compris, cité et recommandé par les moteurs de réponse génératifs** (ChatGPT, Gemini, Perplexity, Copilot...) — au-delà du seul référencement classique sur Google.

Le contenu du site est rédigé pour répondre **directement et explicitement** aux questions qu'un utilisateur poserait à une IA :

- *« Qui fabrique des échappements sur mesure en Alsace ? »*
- *« Où trouver un spécialiste de la soudure TIG inox près de Strasbourg ? »*
- *« Combien coûte une ligne d'échappement sur mesure ? »*

**Zones desservies** (déclarées en `areaServed` dans le schema `Organization`) :

<div align="center">

| 🏠 Siège | 🚗 25 min | 🚗 40 min | 🚗 45 min | 🚗 55 min | 🚗 1h |
|---|---|---|---|---|---|
| Rountzenheim-Auenheim | Haguenau | Strasbourg | Saverne | Sélestat | Colmar |

</div>

**Leviers d'optimisation GEO mis en place :**

- 🧠 **Contenu expert structuré** — sections `WhyChooseSection` et `ExpertiseSection` rédigées comme des réponses directes et autonomes (fabrication, soudure TIG, matériaux, sonorité, processus)
- 📌 **`knowsAbout` schema.org** — déclaration explicite du champ d'expertise pour les crawlers sémantiques
- 🗺️ **Maillage géographique réel** — villes et distances réelles, pas de zone de chalandise générique
- 🔗 **Maillage interne dense** — chaque page renvoie vers les pages contextuellement liées (services ↔ réalisations ↔ contact)
- ❓ **FAQ en langage naturel** — formulée comme des requêtes conversationnelles, pas comme des mots-clés

## 📱 Responsive

Le site a fait l'objet d'un **audit responsive complet**, automatisé via Playwright, sur l'ensemble des routes et des tailles d'écran critiques.

<div align="center">

| 📱 Mobile | 📱 Mobile | 📱 Mobile | 📱 Mobile | 📱 Mobile | 📲 Tablette | 📲 Tablette | 🖥️ Desktop |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| `320px` | `360px` | `375px` | `390px` | `430px` | `768px` | `1024px` | `1440px` |

</div>

**Résultat de l'audit (8 pages × 8 viewports) :**

- ✅ **0 overflow horizontal** détecté sur l'ensemble des combinaisons
- ✅ Header adaptatif — la navigation desktop ne s'active qu'à partir de `1024px`, évitant tout chevauchement en tablette portrait
- ✅ Grilles et cartes repassent proprement en une colonne sur mobile
- ✅ Formulaires entièrement utilisables au doigt (champs, selects avec chevrons, case RGPD)
- ✅ Typographie fluide (`clamp()` / breakpoints `text-*`) — aucun titre tronqué

## ⚙️ Performance

- 🖼️ **Optimisation des polices** — chargement via `next/font/google` (Inter + Oswald), `display: swap`, préconnexions DNS
- 🌀 **Turbopack** — builds et rafraîchissements de développement quasi instantanés
- 🧩 **Server Components par défaut** — le JavaScript client est réservé aux composants réellement interactifs (`"use client"` ciblé)
- 🎯 **Animations optimisées** — `Framer Motion` avec `useInView` (déclenchement uniquement au scroll) et respect de `prefers-reduced-motion`
- 📦 **Génération statique** — 30 routes pré-rendues (`SSG`) à la construction, API routes en dynamique ciblé
- 🧵 **Canvas performant** — animations de particules (Hero, WeldingIntro) via `requestAnimationFrame`, désactivées si mouvement réduit préféré
